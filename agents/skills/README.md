# Runtime skills

Only three skills compete for runtime activation:

| Skill | Use |
| --- | --- |
| `domain-library-ingest-pipeline` | Canonical phase workflow, gates, evidence rules, and stop conditions |
| `domain-library-run-and-operate` | State-driven day-to-day operation |
| `GLM-OCR` | OCR API behavior and response contract |

Architecture, maintenance, diagnostics, research, and retired wiki doctrine live under `docs/maintainers/` so they remain available without confusing runtime routing.
